docker官方文档

一、 安装及加速

安装适用于ubuntu,其他系统请参考官方文档。基础配置及加速通用。

# 安装（根据系统自行选择方式）
sudo apt install docker.io

# 查看版本
docker -v

# 配置docker镜像加速
# 阿里云镜像获取地址：https://cr.console.aliyun.com/cn-hangzhou/instances/mirrors，登陆后，左侧菜单选中镜像加速器就可以看到你的专属地址了
# 按照页面的操作文档配置加速
sudo mkdir -p /etc/docker

sudo tee /etc/docker/daemon.json <<-'EOF'
{
  "registry-mirrors": ["https://改为你的加速器地址.mirror.aliyuncs.com"]
}
EOF

sudo systemctl daemon-reload

sudo systemctl restart docker

# 下载 Docker 图形化界面 portainer
sudo docker pull portainer/portainer

# 创建 portainer 容器
sudo docker volume create portainer_data

# 运行 portainer(需要9000端口没有被占用，如被占用，自行更换第一个宿主机端口)
sudo docker run -d -p 9000:9000 --name portainer --restart always -v /var/run/docker.sock:/var/run/docker.sock -v portainer_data:/data portainer/portainer

# 浏览器输入ip:9000登录图形管理界面

二、 基础命令

2.1 安装、启动及停止

# 安装命令：
apt-get install docker.io

# 关闭docker服务，命令：
service docker stop

# 启动docker 服务，命令：
service docker start

2.2 常用命令

# 查看docker版本，命令：
docker version

# 搜索可用docker镜像，命令：如搜索mysql
docker search mysql

# 下载容器镜像，命令：如拉取mysql镜像
docker pull mysql

# 查看docker镜像及镜像id，命令：
docker images

# 运行一个容器，命令：基于mysql镜像运行一个容器(需要3310端口没有被占用，如被占用，自行更换第一个宿主机端口)
docker run --name firstMysql -p 3310:3306 -e MYSQL\_ROOT\_PASSWORD=123456 -d mysql

跳转到docker run的命令参数详解

# 查看在运行的容器，及容器id，命令：
docker ps

# 查看所有容器（包括未运行的容器）及容器id，命令:
docker ps -a

# 停止一个容器，命令：
docker stop 容器id

# 删除一个容器（正在运行的容器，必须先停止才能删除），命令：
docker rm 容器id

# 删除一个镜像（镜像下有容器，必须先删除容器，才能删除镜像），命令：
docker rmi 镜像id

# 启动所有容器，命令：
docker start $(docker ps -a | awk '{ print $1}' | tail -n +2)

# 关闭所有容器，命令：
docker stop $(docker ps -a | awk '{ print $1}' | tail -n +2)

# 删除所有容器，命令：
docker rm $(docker ps -a | awk '{ print $1}' | tail -n +2)

# 删除所有镜像（先关闭所有的容器，再删除所有容器，才能删除所有镜像），命令：
docker rmi $(docker images | awk '{print $3}' |tail -n +2)

# 查看镜像详细信息，命令：
docker inspect 镜像id

# 进入运行的容器修改相关参数
docker exec -it 容器名称 bash
# 其中bash为进入容器后执行的命令。
# 其中  -d :分离模式: 在后台运行
#       -i :即使没有附加也保持STDIN 打开
#       -t :分配一个伪终端

三、 进阶命令

3.1 docker run命令参数

Usage: docker run [OPTIONS] IMAGE [COMMAND] [ARG...]

-d, --detach=false     指定容器运行于前台还是后台，默认为false
-i, --interactive=false    打开STDIN，用于控制台交互
-t, --tty=false            分配tty设备，该可以支持终端登录，默认为false
-u, --user=""              指定容器的用户
-a, --attach=[]            登录容器（必须是以docker run -d启动的容器）  
-w, --workdir=""           指定容器的工作目录
-c, --cpu-shares=0         设置容器CPU权重，在CPU共享场景使用
-e, --env=[]               指定环境变量，容器中可以使用该环境变量
-m, --memory=""            指定容器的内存上限
-P, --publish-all=false    指定容器暴露的端口
-p, --publish=[]           指定容器暴露的端口
-h, --hostname=""          指定容器的主机名
-v, --volume=[]            给容器挂载存储卷，挂载到容器的某个目录
--volumes-from=[]          给容器挂载其他容器上的卷，挂载到容器的某个目录  
--cap-add=[]               添加权限，权限清单详见：http://linux.die.net/man/7/capabilities
--cap-drop=[]              删除权限，权限清单详见：http://linux.die.net/man/7/capabilities
--cidfile=""               运行容器后，在指定文件中写入容器PID值，一种典型的监控系统用法
--cpuset=""                设置容器可以使用哪些CPU，此参数可以用来容器独占CPU
--device=[]                添加主机设备给容器，相当于设备直通
--dns=[]                   指定容器的dns服务器
--dns-search=[]            指定容器的dns搜索域名，写入到容器的/etc/resolv.conf文件
--entrypoint=""            覆盖image的入口点
--env-file=[]              指定环境变量文件，文件格式为每行一个环境变量
--expose=[]                指定容器暴露的端口，即修改镜像的暴露端口
--link=[]                  指定容器间的关联，使用其他容器的IP、env等信息
--lxc-conf=[]              指定容器的配置文件，只有在指定--exec-driver=lxc时使用
--name=""                  指定容器名字，后续可以通过名字进行容器管理，links特性需要使用名字
--net="bridge"             容器网络设置:  
                              bridge 使用docker daemon指定的网桥
                              host    //容器使用主机的网络
                              container:NAME_or_ID  >//使用其他容器的网路，共享IP和PORT等网络资源
                              none 容器使用自己的网络（类似--net=bridge），但是不进行配置
--privileged=false         指定容器是否为特权容器，特权容器拥有所有的capabilities
--restart="no"             指定容器停止后的重启策略:  
                              no：容器退出时不重启
                              on-failure：容器故障退出（返回值非零）时重启
                              always：容器退出时总是重启
--rm=false                 指定容器停止后自动删除容器(不支持以docker run -d启动的容器)
--sig-proxy=true           设置由代理接受并处理信号，但是SIGCHLD、SIGSTOP和SIGKILL不能被代理
