hexo 多终端更新博客

感谢AndyVJ的指导，原文地址hexo 迁移更换电脑，或多电脑终端更新博客

一、本地 Node.js 和 Git 安装

官网下载安装：

  • 安装 Node.js；
  • 安装 Git。

二、gitHub 新建分支

分支名命名为hexo。

三、设置分支为默认仓库

分支仓库->Settings->Branches->Default branch中将默认分支设为hexo，Update保存。

四、clone 至本地

本地新建一个文件夹用来管理博客，将分支克隆至本地。
git clone https://github.com/你的ID/你的ID.github.io.git

cd进入 clone 下来的你的ID.github.io文件夹，在此文件夹目录下git bash（即命令行）执行git branch命令，新建的分支名 应该是hexo。

五、新电脑生成ssh key并添加至 gitHub

将新电脑生成的ssh key添加到gitHub账户上。
需要配置 git。

  1. git config --global user.email "xxx@qq.com" 引号内的xxx@qq.com改为你github注册并且验证的邮箱；
  2. git config --global user.name "xxxx" 引号内的xxxx改为你的 github 用户名；
  3. ssh-keygen 会出现下面的内容，一直按Enter键就行；
  4. 打开用户目录下的 .ssh目录下面生成 id_rsa(私钥)id_rsa.pub(公钥)两个文件，打开id_rsa.pub，复制里面的内容,到 github: Settings -> SSH and GPG keys -> SSH keys -> New SSH key -> (填写)Title -> (粘贴)Key -> Add SSH Key；
  5. 测试 ssh -T git@github.com，输出 You’ve successfully authenticated 表示添加key 成功。

六、安装 hexo

npm install hexo-cli -g

七、将原博客部分文件复制至你的ID.github.io文件夹下

_config.yml，theme/，source/，scaffolds/，package.json，.gitignore，共6 个文件/文件夹 需要拷贝。

需要备份的文件如下:

  1. _config.yml（站点配置）；
  2. theme文件夹里面的主题；
  3. source文件夹；
  4. scaffolds文件夹（文章的模板）；
  5. package.json（说明使用哪些包）；
  6. .gitignore（限定在提交的时候哪些文件可以忽略）；

注意：不要hexo init去整体初始化，因为需要的文件我们已经拷贝过来了。

九、提交本地至hexo分支

cd进入你的ID.github.io文件目录下，依次执行：

  1. git add .
  2. git commit -m "新电脑部署"（引号内容可改）
  3. git push

这样，master分支用于保存博客静态资源，提供博客页面供人访问；hexo分支用于备份博客部署文件，供自己修改和更新，两者在一个GitHub仓库内互不冲突。

十、安装 hexo 依赖的包

在你的ID.github.io文件目录下，执行：

npm install
如果出现报错ERROR Try runing: 'npm install hexo --save则执行命令npm install hexo --save，如果没有，则忽略。

最后：多终端更新博客

  1. git pull；
  2. hexo n xxx 编辑、撰写文章或其他博客更新改动，就是你要对博客进行的修改，或新增文章；
  3. hexo clean // 可选；
  4. hexo g；
  5. hexo s；
  6. hexo d。
  7. git add .；
  8. git commit -m "在新电脑上提交新文章"（引号内容可改）;
  9. git push指令，保证hexo分支版本为最新版本。

从上述第 3 步后可以这样子发布：
hexo clean && hexo g && hexo d && git add . && git commit -m "更新" && git push && exit
把命令用&&串起来，有一个失败就会停止；成功，最后一条命令就会退出终端。
